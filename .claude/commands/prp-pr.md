# PRP PR Command

Phase 4 of the Product Requirements Process. Creates a GitHub pull request from the current branch, auto-discovering any PR template and generating a description from commit history and the PRP plan (if available).

## Mandatory Skills

- **using-superpowers** - Core capabilities
- **git-workflow** - Branch and PR operations

## Task

Create PR: $ARGUMENTS

---

## Execution Steps

### Step 1: Pre-flight Checks

```bash
# Verify on a feature branch (not main/master)
BRANCH=$(git branch --show-current)
if echo "$BRANCH" | grep -qE "^(main|master|develop)$"; then
    echo "ERROR: Cannot create PR from $BRANCH. Create a feature branch first."
    exit 1
fi

# Check if branch has commits ahead of main
AHEAD=$(git rev-list main..HEAD --count 2>/dev/null || git rev-list master..HEAD --count 2>/dev/null)
if [ "$AHEAD" -eq 0 ]; then
    echo "ERROR: No commits ahead of main. Nothing to PR."
    exit 1
fi

echo "Branch: $BRANCH ($AHEAD commits ahead of main)"
```

### Step 2: Gather Context

```bash
# Commit history for this branch
git log main..HEAD --oneline

# Files changed
git diff main..HEAD --stat | tail -5

# Check for PR template
ls .github/pull_request_template.md .github/PULL_REQUEST_TEMPLATE.md \
   .github/PULL_REQUEST_TEMPLATE/*.md 2>/dev/null | head -3

# Check for recent PRP plan
ls -t .claude/plans/prp-*.md 2>/dev/null | head -1
```

### Step 3: Generate PR Title

If `--title` not provided, generate from the most descriptive commit:
- Use the commit message of the first commit on this branch
- Or derive from the PRP plan goal (if found)
- Format: conventional commit style, ≤ 70 chars

### Step 4: Generate PR Body

Use PR template if found, otherwise use this structure:

```markdown
## Summary
- <bullet summarizing what this PR does>
- <key decision made>
- <any breaking changes>

## Changes
<auto-generated from git diff --stat>

## Test Plan
- [ ] Unit tests pass (`<test command>`)
- [ ] Integration tests pass
- [ ] Manual verification: <what to test manually>

## Related
<PRP plan: .claude/plans/prp-*.md if found>

🤖 Generated with [ClaudeKit](https://github.com/your-org/claudekit)
```

### Step 5: Create PR

```bash
DRAFT_FLAG=""
echo "$ARGUMENTS" | grep -q '\-\-draft' && DRAFT_FLAG="--draft"

gh pr create \
  --title "<generated title>" \
  --body "$(cat <<'EOF'
<generated body>
EOF
)" \
  $DRAFT_FLAG
```

### Step 6: Confirm

```
PR CREATED
==========
URL: <pr url>
Title: <title>
Branch: <branch> → main
Commits: <N>
Files: <N> changed

Next steps:
  - Review the PR in GitHub
  - Run /santa <files> for dual review (recommended for auth/security changes)
  - Merge when approved
```

---

## Usage Examples

- `/prp-pr` — create PR with auto-generated title and body
- `/prp-pr --title "feat(auth): add JWT refresh token rotation"` — explicit title
- `/prp-pr --draft` — create as draft PR
- `/prp-pr --title "fix: race condition in order processor" --draft` — draft with title

## Notes

- Requires `gh` (GitHub CLI) — install with `brew install gh` or `apt install gh`
- Auto-discovers `.github/pull_request_template.md` if present
- Uses PRP plan (if found) to populate the description
- NEVER creates a PR from `main`, `master`, or `develop`
